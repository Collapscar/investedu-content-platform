# 把平台从"能管理内容"升级为"能一眼看出内容资产价值"

## Context

当前 MVP 已收敛到 3 个模块（内容门户 / 素材库 / 专题·交付），框架对了，但**内容资产本身没有被真正呈现出来**：

- 内容门户默认选中"基金"频道，但该频道 0 专题，第一屏是空状态 —— 对银行客户是危险的第一印象。
- 素材封面与详情都是纯蓝色占位图，看起来像 Demo，不像内容资产平台；核心资产（投教长图）反而最不显眼。
- 素材卡片上没有"加入专题"等闭环动作，看完不知道下一步去哪。
- 专题/交付页面像列表，没有"快速把素材变成可投放专题"的流程感。
- 命名 `InvestEd Studio` 偏通用创作工具，不像卖给银行的投教资产服务。
- 素材库标题、筛选计数、专题计数概念混淆（128 vs 12）。

本计划聚焦把这些问题逐项修掉，让银行打开平台第一屏就能感受到"我买到了一套可直接使用的投教内容资产"。

---

## 设计原则

- **不加新模块**，只在 3 个现有页面里把"内容资产"做实。
- **真实图替占位图**：使用已注册的 Unsplash MCP（`mcp__plugin_make_unsplash__search_photos`）按频道主题取图，套到所有长图封面与专题封面位上。
- **真实文本**：长图详情用真实标题、客户问题、摘要、风险提示文本（已有数据基本可用，再补强 1–2 字段）。
- **默认显示最完整样板**：默认频道、默认选中专题、默认选中素材都指向已经完整的"养老"链路。
- **复用已有组件**：`src/app/components/figma/ImageWithFallback.tsx`（标准图片组件，带占位回退）、shadcn 卡片 / 徽章 / 分隔线 / 标签栏。

### 2. `src/app/components/pages.tsx` —— 数据与展示

#### 2.1 素材数据补强（顶部 `ASSETS` 数组）

- 给每个 `Asset` 增加两个字段：
  - `cover: string` —— 真实长图封面 URL（Unsplash，按频道选题）。
  - `risk: string` —— 单独的风险提示文本（当前页面已写死一段通用文本，改为按品类微调，让详情看起来不像 demo）。
- 频道→Unsplash 关键词建议：
  - 养老 → `retirement planning`, `elderly couple finance`
  - 基金 → `mutual fund chart`, `investing learn`
  - 理财 → `personal finance`, `savings`
  - 黄金 → `gold bar`, `gold investment`
  - 信贷 → `bank card`, `consumer loan`
  - 保险 → `insurance umbrella`
  - 代发 → `payroll`, `salary card`
- 取图方式：调用 `mcp__plugin_make_unsplash__search_photos`，把返回 URL 直接写进 `ASSETS[*].cover`（不走运行时网络，落到源代码里，便于版本固定）。
- 在专题渲染处，专题封面 = `assetIds[0]` 的封面（自动派生，不另存）。

#### 2.2 用 `ImageWithFallback` 替换所有纯蓝色 placeholder

- 当前所有 `aspect-[3/4]`、`aspect-[16/9]` 的 `<div bg-gradient-to-br from-primary…>` 都改为：  
  `<ImageWithFallback src={asset.cover} alt={asset.title} className="size-full object-cover" />`
- 位置：
  - 素材库网格卡片
  - 素材详情右侧大图（改成 `aspect-[3/4]` 真实长图预览）
  - 专题/交付列表（如有缩略图位）
  - 专题详情素材清单缩略图
  - 内容门户手机预览专题卡片封面 + 专题侧栏素材清单
- 文件路径：`./figma/ImageWithFallback`。

#### 2.3 内容门户（`PortalPage`）

- **默认频道改为 `养老`**（已有 3 个发布专题）。同时把"如果当前频道无已发布专题就回退到第一个有专题的频道"逻辑写到组件初始化里，永远不留空状态作为第一屏。
- 真正没有专题的频道，中部手机壳内提示文案 + 双按钮：  
  `本频道暂无已发布专题` →  `[去素材库选内容]` `[新建本频道专题]` —— 两个按钮分别跳 `assets` / `topics(create)`。
- 手机预览顶部「{channel}专区」下补一句固定文案：  
  `回答客户关心的真实问题` —— 已存在，保留。  
  下方专题卡片至少展示 2 个真实专题（养老频道天然 3 个）。
- 右栏「当前专题」固定显示 H5 链接 + 二维码 + 复制链接 3 件套，**移除"专题包含素材"列表里的纯文字行**，改为真实小缩略图 + 标题。

#### 2.4 素材库（`AssetsPage`）

- 顶部计数文案改：`128 件素材` + 右侧 `当前筛选 12 件`（之前只显示 `共 12 件`）。
- 卡片固定操作三选项（移除原先的"复制 / 历史 / 下载"图标组）：

  - `加入专题`（主按钮，蓝色）
  - `下载审核包`（次按钮）

  1) 长图大预览（真实图）  
  2) 客户问题  
  3) 摘要

  删掉「品类 / 状态」DetailRow（顶部 Badge 已显示）。

#### 2.5 专题 / 交付（`TopicsPage` + `TopicCreate` + `TopicDetail`）

  三栏继续保留，但每栏标题前加 `Step N` 编号；底部主按钮改为 `生成 H5 链接与二维码`。
- 专题详情（`TopicDetail`）的素材排行/清单里也用真实封面缩略图。

#### 2.6 文案微调

- TopBar 的 `subtitle` 在 3 个页面统一改：

## 验证方式（端到端走查）

1. **首屏不空**：刷新进入"内容门户" → 默认频道为「养老」→ 中部手机壳里能看到 ≥2 张真实封面的专题卡片，右栏显示真实 H5 + 二维码。
2. **空频道有出口**：切到"基金" → 中部出现引导卡：`[去素材库选内容]` / `[新建本频道专题]`，两个按钮可点并正确跳转。
3. **素材闭环**：进入「素材库」→ 卡片显示真实长图封面 → 点 `加入专题` → 自动跳到「专题/交付 → 新建专题」并预填该素材 → Step 进度条显示在第 2 步。
4. **真实长图预览**：素材库右栏素材详情上方显示真实长图大图（非纯色块），下方风险提示、加入专题、审核包按钮齐全。
5. **专题流程感**：列表顶部 Banner 突出"三步生成"信息；点 `新建专题` 后顶部出现 3 步进度条；走完 Step 3 后跳到专题详情，详情页能复制 H5、下载二维码。
6. **计数清晰**：素材库 TopBar 显示 `128 件素材`，工具栏右侧显示 `当前筛选 N 件`；侧边栏素材库 hint = 128，专题/交付 hint = 已发布数。
7. **命名生效**：侧边栏品牌名为「金融投教内容承接平台」，无 `InvestEd Studio` 字样，无任何具体银行名称。
