# 金融投教内容资产平台 UI 设计规范

> 版本：MVP 1.0  
> 适用对象：银行内容运营、品牌、消费者权益保护人员  
> 产品定位：已采购金融投教内容资产的查询、理解、预览与领取平台  
> 视觉参考：火山引擎企业级产品的蓝白科技感、清晰层级与模块化布局  
> 关联文档：[平台设计规范-火山引擎参考.md](./平台设计规范-火山引擎参考.md)

## 1. 设计目标

平台 UI 必须优先帮助银行用户完成以下任务：

1. 看清本机构已采购哪些内容资产。
2. 通过客户问题和使用场景，在 30 秒内找到适合内容。
3. 理解每篇素材适合何时使用、如何使用以及使用边界。
4. 下载包含图片、文案、说明和风险提示的完整交付包。
5. 理解专题包中的素材组合逻辑，并一次领取全部交付物。

### 1.1 设计原则

- **资产视角**：页面强调“已采购、当前可用、覆盖问题、可直接领取”，不强调生产效率。
- **客户问题优先**：标题、客户问题和适用场景是查找内容的主要入口，品类只是辅助分类。
- **下载优先**：主操作始终是下载完整交付包，预览为次操作。
- **完整交付**：不得让 PNG 单图看起来是主要或唯一交付物。
- **只读可信**：银行端不出现新建、编辑、审核、发布、下架和维护入口。
- **真实可判断**：所有内容卡片使用真实长图缩略图，不使用占位封面或抽象图标替代内容。
- **合规可理解**：风险提示、免责声明和不建议使用场景必须容易找到，而非隐藏在下载文件中。

## 2. 信息架构与权限

### 2.1 一级导航

左侧只保留三个一级导航：

1. 内容资产
2. 素材库
3. 专题包

导航不折叠二级菜单。点击 Logo 返回“内容资产”。

### 2.2 银行端允许操作

- 搜索、筛选、排序和清空筛选。
- 查看资产范围和内容说明。
- 预览素材、专题包和可选 H5。
- 复制推荐转发文案或 H5 链接。
- 单篇或批量下载素材包。
- 下载完整专题包、二维码和 PNG 原图。

### 2.3 银行端禁止出现

- 新建、编辑、上传、删除、调整顺序。
- 提交审核、审核通过、驳回。
- 发布、撤回、下架、恢复。
- 加入专题包、创建专题包。
- 版本对比、需求工单、渠道管理。
- 访问量、下载热度、转化率和用户行为追踪。

禁止项不仅需要禁用，而是不得渲染入口，避免形成错误权限预期。

## 3. 全局框架

### 3.1 页面尺寸

| 项目 | 规范 |
| --- | --- |
| 设计基准 | `1440 × 900px` |
| 最小支持桌面宽度 | `1024px` |
| 顶部栏高度 | `56px` |
| 左侧导航宽度 | `208px` |
| 内容区左右边距 | `24px`，大于 1600px 时使用 `32px` |
| 内容区最大宽度 | `1600px` |
| 页面标题区 | `72–88px` |
| 模块纵向间距 | `24px` |
| 卡片间距 | `16px` |

平台是桌面工作平台。移动端仅保障搜索、预览和单项下载，不要求支持批量领取和复杂筛选。

### 3.2 顶部栏

从左到右依次为：

1. 平台 Logo 和产品名。
2. 全局搜索框。
3. 帮助入口。
4. 当前银行名称和账户菜单。

全局搜索框宽 `480–560px`，占位文案：

> 搜索素材标题、客户问题或专题包名称

删除通知铃铛。顶部栏不放“新建”“上传”等操作。

### 3.3 左侧导航

- 导航背景：`#FFFFFF`，右侧边框 `#E5E6EB`。
- 菜单项高度：`44px`，左右内边距 `16px`。
- 当前项：浅蓝底 `#E8F3FF`、主蓝文字、左侧 `3px` 主蓝指示条。
- 图标：20px 线性图标，文字 14px/500。
- 导航底部可显示“内容服务由平台方提供”，不得放运营管理入口。

## 4. 视觉基础

### 4.1 色彩

| Token | 色值 | 用途 |
| --- | --- | --- |
| `primary` | `#1664FF` | 下载按钮、链接、选中状态 |
| `primary-hover` | `#0E42D2` | 主按钮悬停 |
| `primary-light` | `#E8F3FF` | 选中背景、轻提示 |
| `text-1` | `#1D2129` | 页面标题、内容标题 |
| `text-2` | `#4E5969` | 正文、客户问题、说明 |
| `text-3` | `#86909C` | 时间、编号、辅助信息 |
| `border` | `#E5E6EB` | 卡片、输入框、分割线 |
| `fill` | `#F2F3F5` | 标签、筛选区域、禁用背景 |
| `page-bg` | `#F7F8FA` | 页面背景 |
| `surface` | `#FFFFFF` | 卡片、抽屉、弹窗 |
| `success` | `#00B42A` | 可用 |
| `warning` | `#FF7D00` | 建议更新、准备更新 |
| `updated` | `#168CFF` | 已更新 |
| `danger` | `#F53F3F` | 下载失败、文件异常 |

主色仅用于可操作和被选中内容。品类不使用七种高饱和颜色，而使用统一浅灰或浅蓝标签。

### 4.2 状态规范

| 状态 | 样式 | 使用位置 |
| --- | --- | --- |
| 可用 | 绿字浅绿底 + 对勾图标 | 素材卡、素材详情 |
| 建议更新 | 橙字浅橙底 + 提示图标 | 素材卡、素材详情 |
| 已更新 | 蓝字浅蓝底 + 更新图标 | 最新交付、更新服务 |
| 准备更新 | 橙色状态点 + 文案 | 内容更新服务 |

- 标签高度 `24px`，左右内边距 `8px`，圆角 `4px`。
- 状态必须同时有文字与图形，不依赖颜色识别。
- 银行端不出现草稿、审核中、已下架等平台内部状态。

### 4.3 字体层级

| 层级 | 字号/行高 | 字重 | 用途 |
| --- | --- | --- | --- |
| H1 | `28/40` | 600 | 页面标题 |
| H2 | `22/32` | 600 | 首页模块、详情大区块 |
| H3 | `18/28` | 600 | 卡片组、详情段落 |
| Body | `14/22` | 400 | 默认正文 |
| Body Small | `13/20` | 400 | 卡片辅助信息 |
| Caption | `12/18` | 400 | 日期、数量、编号 |
| Data | `30/38` | 600 | 资产概览数字 |

中文字体使用 `PingFang SC`、`Microsoft YaHei`；数字使用 `Inter` 并启用等宽数字。

### 4.4 卡片与阴影

- 普通卡片：白底、`8px` 圆角、`1px solid #E5E6EB`。
- 信息卡内边距：`20px`；素材卡正文内边距：`16px`。
- 默认无阴影；悬停时边框变为 `#94BFFF`，增加轻阴影。
- 不使用玻璃拟态、强发光、悬浮大渐变等与资产平台不匹配的装饰。

## 5. 核心组件规范

### 5.1 按钮

| 层级 | 样式 | 固定文案 |
| --- | --- | --- |
| 主按钮 | 蓝底白字 | 下载素材包、下载专题包、下载完整专题包 |
| 次按钮 | 白底蓝字、蓝色边框 | 预览、预览专题、预览素材 |
| 文字按钮 | 蓝色文字 | 查看素材、查看专题包、查看当前版本 |
| 辅助按钮 | 白底灰边 | 清空筛选、取消选择、复制文案 |

- 默认高度 `36px`，详情主交付按钮使用 `40px`。
- 所有下载按钮包含下载图标和明确下载对象。
- 禁止仅使用无文字下载图标。
- 下载 H5 二维码属于辅助操作，不使用主按钮样式。

### 5.2 全局搜索

搜索建议面板分为三组：

1. 客户问题建议。
2. 相关素材。
3. 相关专题包。

每条建议显示匹配字段，如“客户问题匹配”。键盘支持上下选择、回车搜索和 Esc 关闭。

搜索提交后进入全局搜索结果页，不在浮层中承载全部结果。

### 5.3 筛选栏

- 搜索框位于第一行左侧，宽度 `320–400px`。
- 品类、使用场景、内容状态使用下拉多选。
- 已选筛选条件在第二行以可关闭标签展示。
- 右侧固定显示“当前结果 25 件”和“清空全部筛选”。
- 筛选后即时刷新结果，不额外设置“确认筛选”按钮。
- 结果数量表达当前结果，不使用“共 25/采购 25”混合口径。

### 5.4 真实长图缩略图

- 素材卡缩略图区比例固定为 `3:4`，使用真实长图顶部区域生成缩略图。
- 图片采用 `object-fit: cover; object-position: top center;`。
- 缩略图右下角显示“长图”标识，提示可以滚动查看完整内容。
- 不使用产品图标、纯色背景或“暂无封面”作为正常状态。
- 图片加载失败显示错误态：“预览图加载失败”，保留标题和下载操作，并上报异常。
- 图片懒加载时使用与真实封面主色接近的模糊预览，不使用通用灰色骨架长期占位。

### 5.5 素材卡片

建议桌面端四列布局：

- `>= 1440px`：4 列。
- `1200–1439px`：3 列。
- `1024–1199px`：3 列，压缩卡片间距。

卡片信息顺序：

1. 多选框，位于封面左上角。
2. 真实长图缩略图。
3. 品类标签和状态标签。
4. 素材标题，最多两行。
5. 客户问题，前缀固定为“客户问题：”，最多两行。
6. 使用场景标签，最多显示两个，其余以 `+N` 表示。
7. 更新时间。
8. `下载素材包`主按钮和`预览`次按钮。

卡片不得出现加入专题、编辑、状态修改、删除和无说明图标按钮。

### 5.6 专题包卡片

专题列表采用纵向列表卡或两列宽卡，不使用与单篇素材完全相同的窄卡。

每项结构：

1. 左侧专题封面，比例 `16:9`。
2. 专题名称。
3. 一句话业务说明，最多两行。
4. 适用场景、适用对象。
5. 包含素材数量、最近更新时间。
6. `下载专题包`主按钮和`预览`次按钮。

专题封面应体现使用场景和组合内容，不使用下载次数或热度角标。

### 5.7 下载反馈

单项下载流程：

1. 点击下载按钮。
2. 按钮进入“正在准备文件…”状态。
3. 文件生成完成后自动开始下载。
4. Toast 提示“素材包已开始下载，包含 5 项文件”。

批量下载流程：

1. 用户勾选素材，页面底部出现吸附式批量操作栏。
2. 点击“批量下载素材包”。
3. 弹窗说明将生成一个 ZIP，每篇素材保持独立文件夹。
4. 生成期间展示进度和已处理数量。
5. 失败时保留选择，并提供重新生成。

文件名格式：

```text
素材包_素材编号_素材标题_YYYYMMDD.zip
专题包_专题名称_YYYYMMDD.zip
批量素材包_N篇_YYYYMMDD.zip
```

## 6. 页面一：内容资产

### 6.1 页面结构

页面按以下顺序组成：

1. 页面标题与采购范围说明。
2. 资产概览。
3. 品类覆盖。
4. 最新交付素材。
5. 内容更新服务。
6. 推荐专题包。

页面标题下建议说明：

> 查看本机构已采购的内容范围、最新交付和可直接使用的专题方案。

### 6.2 资产概览

四张指标卡横向排列：

| 指标 | 数值 | 点击行为 |
| --- | --- | --- |
| 已采购素材 | 25 件 | 进入素材库全部内容 |
| 当前可用素材 | 23 件 | 进入素材库，并选中“可用” |
| 覆盖客户问题 | 25 个 | 打开客户问题目录抽屉或结果页 |
| 标准专题包 | 10 个 | 进入专题包列表 |

指标卡规范：

- 高度 `112px`，内边距 `20px`。
- 数字 30px/600，单位 14px。
- 卡片底部显示明确入口文案，如“查看全部素材 →”。
- Hover 强调边框，整卡可点击且具备键盘焦点。
- 不展示访问量、增长率和下载次数。

### 6.3 品类覆盖

标题右侧可显示“采购范围内共 7 个品类”。

每个品类卡展示：

- 品类名称。
- `素材 5 篇 · 专题包 2 个`。
- 覆盖主题，最多两行。
- 最近更新时间。
- `查看素材`次按钮。
- `查看专题包`文字入口。

推荐使用三列卡片布局。只展示已采购品类，排序由采购范围固定，不根据使用量动态改变。

### 6.4 最新交付素材

- 展示 3–5 项，优先使用横向列表卡，便于呈现长图缩略图和交付日期。
- 每项显示缩略图、标题、品类、交付日期和“已更新/可用”状态。
- 操作为`下载素材包`和`预览`。
- 不提供独立 PNG 图标按钮；PNG 原图下载只出现在素材详情。

### 6.5 内容更新服务

使用白色列表卡，不使用警告大红色区域。

列信息：

1. 当前素材标题。
2. 更新原因。
3. 平台处理状态。
4. 预计提供时间或更新时间。
5. 查看当前版本。

文案示例：

- 更新原因：引用政策发生变化。
- 状态：准备更新。
- 预计提供：2026 年 6 月 30 日前。

不得出现“请银行修改”“待银行处理”等责任指向错误的文案。

### 6.6 推荐专题包

建议使用两列宽卡，每张卡展示：

- 专题名称。
- 一句话说明解决的问题。
- 适用场景。
- 包含素材数。
- `下载专题包`和`预览专题`。

推荐依据使用场景和采购范围，不显示热度或下载次数。

## 7. 页面二：素材库

### 7.1 页面头部

- 标题：素材库。
- 说明：查找本机构已采购、可直接领取使用的金融投教素材。
- 页面头部不设置新建或上传按钮。

### 7.2 搜索与筛选

筛选项：

- 搜索：标题、客户问题、关键词。
- 品类：基金、理财、黄金、信贷、保险、养老、代发。
- 使用场景：客户咨询、主题宣传、风险教育、节日活动。
- 内容状态：可用、建议更新。

默认查询条件为当前机构采购范围，界面不额外展示不可修改的“已采购”筛选项。

搜索无结果时展示：

1. 当前搜索词和已选筛选条件。
2. “未找到完全匹配的素材”。
3. 2–4 个相近客户问题。
4. 相邻品类入口。
5. 清空筛选按钮。

### 7.3 结果区

- 左侧显示“当前结果 25 件”。
- 右侧可切换默认排序和最近更新。
- 素材采用卡片视图，默认每页 20 件。
- 翻页后返回列表顶部，保留全部筛选条件。

### 7.4 批量领取

素材卡左上角设置多选框。选择后底部显示吸附操作栏：

> 已选择 3 件素材　[批量下载素材包] [取消选择]

- 操作栏距浏览器底部 `16px`，宽度随内容区。
- 翻页时保留已选项，并显示跨页总选择数。
- 单次最多选择数量由技术约束决定，达到上限时明确提示。

### 7.5 素材详情

从列表预览时使用右侧大抽屉，推荐宽度 `720–840px`；点击“查看完整详情”进入独立详情页。

抽屉/详情按以下四段排列：

#### 内容预览

- 左侧或顶部为长图滚动容器，高度不超过视口的 `70vh`。
- 提供“查看大图”和“下载 PNG 原图”。
- 大图查看器支持缩放、适应宽度、1:1 和关闭。
- PNG 下载属于预览区次级操作，不替代完整素材包下载。

#### 内容说明

采用标签和值的两列描述列表：

- 标题。
- 内容编号。
- 所属品类。
- 客户问题。
- 核心摘要。
- 适用场景。
- 适用对象。
- 更新时间。

客户问题和核心摘要占整行，其他字段可双列排列。

#### 使用指引

按以下小节展示：

1. 适合什么时候使用。
2. 推荐转发文案，右上角提供`复制文案`。
3. 理财经理讲解提示。
4. 不建议使用的场景或表达。
5. 风险提示和免责声明。

“不建议使用”和“风险提示”使用浅橙提示卡，不使用红色错误态。

#### 领取内容

- 使用浅蓝背景的重点交付卡。
- 主按钮为`下载完整素材包`。
- 按钮下展示文件清单和文件格式。

固定文件清单：

1. 长图 PNG。
2. 内容说明 PDF 或 DOCX。
3. 推荐转发文案 TXT。
4. 风险提示与免责声明 TXT。
5. 文件清单。

## 8. 页面三：专题包列表

### 8.1 页面头部

标题：专题包。

固定说明：

> 平台方围绕具体客户问题和使用场景预制的内容解决方案，可直接下载使用。

页面不出现新建专题包按钮。

### 8.2 筛选栏

- 搜索专题包名称和客户问题。
- 品类筛选。
- 使用场景筛选。
- 排序：默认排序、最近更新。

不设置草稿、已下架筛选，不展示下载次数。

### 8.3 组织方式

默认优先按使用场景排序和推荐，例如：

- 贷前风险教育。
- 基金新手入门。
- 家庭保障规划。
- 养老准备起步。

品类作为标签和筛选条件，不作为唯一的信息组织结构。

### 8.4 列表项

每项展示：

- 专题封面。
- 专题名称。
- 一句话业务说明。
- 适用场景。
- 适用对象。
- 包含素材数量。
- 最近更新时间。
- `下载专题包`。
- `预览`。

整卡可进入专题详情，但下载按钮点击时不得触发页面跳转。

## 9. 页面四：专题包详情

### 9.1 顶部摘要

左侧展示专题名称和业务说明，右侧为重点交付卡。

摘要字段：

- 所属品类。
- 解决的客户问题。
- 适用场景。
- 适用对象。
- 内容目标。
- 包含素材数。
- 更新时间。

右侧交付卡固定显示：

- `下载完整专题包`主按钮。
- 文件数量和预计压缩包大小。
- 文件清单入口。

### 9.2 为什么这样组合

使用横向三步流程卡或纵向步骤条，推荐三步结构：

1. **建立认知**：第一步解决什么认知问题。
2. **辅助判断**：第二步帮助客户完成什么判断。
3. **提示行动与风险**：第三步提示什么风险或行动。

每一步需关联其对应素材编号。该模块是专题包与普通文件集合的关键区别，不能折叠隐藏。

### 9.3 包含素材

按平台预设顺序展示纵向列表：

1. 顺序编号，使用醒目的 `01/02/03`。
2. 真实长图缩略图。
3. 素材标题。
4. 该素材在专题中的作用。
5. 客户问题。
6. `预览素材`。

银行端不得出现拖拽手柄、删除、替换和调整顺序入口。

### 9.4 使用指引

使用分组内容卡展示：

- 推荐使用方式：一次发送或分次发送。
- 推荐发送顺序。
- 配套转发文案，支持一键复制。
- 理财经理讲解边界。
- 风险提示与免责声明。

推荐发送顺序应与包含素材顺序一致；如存在不同策略，需要明确解释适用条件。

### 9.5 交付物

主交付区展示以下固定文件：

1. 全部长图 PNG。
2. 专题目录。
3. 内容使用顺序。
4. 专题介绍。
5. 配套转发文案。
6. 单篇素材说明。
7. 风险提示及免责声明。

主操作只保留`下载完整专题包`。

H5 与二维码放入默认收起的“其他使用方式”：

- 预览 H5。
- 复制 H5 链接。
- 下载二维码。

不得设置编辑专题包和下架入口。

## 10. 全局搜索结果页

### 10.1 页面结构

标题格式：

> “客户担心基金亏损”的搜索结果

页面分为两个明确区域：

1. 相关素材。
2. 相关专题包。

区域标题显示结果数量，可分别查看更多。

### 10.2 匹配说明

每个结果必须展示一项主要匹配依据：

- 标题匹配。
- 客户问题匹配。
- 内容关键词匹配。
- 使用场景匹配。

客户问题匹配权重最高，其次为专题解决问题、标题和内容关键词。高相关结果不得仅因标题没有完全包含搜索词而排在后面。

### 10.3 结果样式

- 素材结果展示真实长图小缩略图、标题、客户问题、状态和下载/预览操作。
- 专题包结果展示专题封面、专题名称、解决问题、包含素材数和下载/预览操作。
- 命中关键词可以高亮，但高亮背景使用浅黄 `#FFF7E8`，不改变正文可读性。

## 11. 空、加载与异常状态

### 11.1 空状态

| 场景 | 文案 | 操作 |
| --- | --- | --- |
| 搜索无结果 | 未找到完全匹配的内容 | 查看相近问题、清空筛选 |
| 采购品类无专题包 | 当前采购范围暂无该场景专题包 | 查看相关素材 |
| 内容更新为空 | 当前没有需要关注的内容更新 | 无主操作 |

不得显示“去创建”“去上传”“联系管理员发布”等不符合银行权限的操作。

### 11.2 加载状态

- 卡片列表使用保持布局稳定的骨架屏。
- 长图预览显示加载进度，加载完成后保持当前位置。
- 下载文件准备时间超过 2 秒时显示明确阶段文案。

### 11.3 异常状态

- 文件生成失败：提示原因和重新下载，不取消用户选择。
- 预览加载失败：允许直接下载完整素材包。
- 无权限或采购范围变化：提示“该内容当前不在本机构可用范围内”，返回素材库。

## 12. 文案规范

### 12.1 推荐用词

- 已采购素材。
- 当前可用素材。
- 下载素材包。
- 下载完整专题包。
- 内容更新服务。
- 平台准备更新。
- 适合什么时候使用。
- 不建议使用的场景或表达。

### 12.2 禁止用词

- 新建内容、立即创作、智能生成。
- 提交审核、发布内容、下架素材。
- 请银行更新、待您修改。
- 爆款、热门、下载排行。
- 一键获客、保证转化、稳赚等不合规表达。

### 12.3 数量和日期

- 素材数量统一使用“件”或“篇”，同一页面不得混用；建议指标用“件”，内容描述用“篇”。
- 专题包数量使用“个”。
- 日期列表统一为 `YYYY-MM-DD`，正文可使用“2026 年 6 月 15 日”。
- 更新时间必须说明对象，如“素材更新时间”，避免只有孤立日期。

## 13. 响应式规则

| 宽度 | 调整方式 |
| --- | --- |
| `>= 1440px` | 四列素材卡，四张指标卡同排 |
| `1200–1439px` | 三列素材卡，四张指标卡同排 |
| `1024–1199px` | 三列紧凑素材卡，专题详情改为单栏 |
| `768–1023px` | 左侧导航收起，素材卡两列，筛选进入抽屉 |
| `< 768px` | 单列列表，仅保障搜索、预览和单项下载 |

长图大图查看器在窄屏中使用全屏模式。

## 14. 可访问性

- 正文与背景对比度不低于 `4.5:1`。
- 所有卡片点击区域、复选框和按钮支持键盘操作。
- 焦点框使用 `2px solid #1664FF`，不得移除默认焦点后不提供替代。
- 长图提供内容标题作为替代文本。
- 下载按钮需要向读屏软件说明对象，例如“下载素材包：基金亏损如何理解”。
- 状态标签使用文字和图标双重表达。
- Toast 停留时间不少于 4 秒，下载失败信息需持续可见。

## 17. UI 验收标准

- [ ] 一级导航只有内容资产、素材库、专题包。
- [ ] 顶部无通知铃铛，搜索支持素材、客户问题和专题包。
- [ ] 所有正常内容卡使用真实长图或真实专题封面。
- [ ] 银行端不存在新建、编辑、上传、审核、发布和下架入口。
- [ ] 页面主操作统一指向完整素材包或专题包下载。
- [ ] 单独 PNG 下载只出现在素材详情的内容预览区。
- [ ] 素材卡完整显示客户问题、场景、状态和更新时间。
- [ ] 筛选结果数量与采购总数不会混淆。
- [ ] 批量下载明确生成一个 ZIP，且每篇素材保持独立文件夹。
- [ ] 素材详情完整展示使用时机、转发文案、讲解提示和使用边界。
- [ ] 专题详情在素材列表前说明“为什么这样组合”。
- [ ] 专题包下载区清楚列出全部交付文件。
- [ ] H5 与二维码放在默认收起的其他使用方式中。
- [ ] 搜索结果展示匹配依据，客户问题匹配优先于标题字面匹配。
- [ ] 无结果、加载失败和下载失败状态均不引导用户创建内容。
- [ ] 1440px、1280px、1024px 下无关键操作遮挡或信息丢失。
